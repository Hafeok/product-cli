# CTX001 — Source file length

Every Rust source file under `src/` and `xtask/src/` must be **at most 400
lines**. Files that exceed the limit fail `cargo xtask check` with diagnostic
code `CTX001`.

## Why

A 400-line cap forces module decomposition along seam lines. When a file
grows past it, the rule turns "I'll refactor later" into a build break, which
is the only deadline that holds. See [ADR-0001](../adr/ADR-0001-file-length-limit.md)
for the full rationale and rejected alternatives.

## Triggers

- Any `.rs` file under `src/` or `xtask/src/` with more than 400 lines.
- Files that fail to parse as valid Rust also surface a diagnostic so the
  scan never silently skips a file.

## How to fix

Split the offending module into focused submodules along an existing seam:

- A pure-vs-effectful seam (`plan_*` / `apply_*` per the project's
  Slice + Adapter pattern).
- A read-vs-write seam (`build_*` / `render_*` for queries, `apply_*` for
  writes).
- A serialization seam (parse, normalize, serialize as separate modules).

If the file is a single large enum or struct definition that legitimately
cannot be split, prefer:

1. Moving the impl blocks into a sibling `impls.rs` module.
2. Pulling derive-heavy serde representations into a `repr.rs` module.

There is no per-file `#[allow(...)]` escape hatch. The limit is the limit.

## Examples

**Triggers CTX001:**

```rust
// src/giant_module.rs — 412 lines
//! Holds twelve unrelated helpers that grew over a year.
// ... 400+ lines ...
```

**Passes CTX001:**

```
src/giant_module/
├── mod.rs        // 80 lines, re-exports + module docstring
├── plan.rs       // 180 lines, pure planning logic
├── apply.rs      // 120 lines, atomic-write side effects
└── render.rs     // 90 lines, text + JSON renderers
```

## Enforcement

- **Tier:** 3b (xtask, syntactic).
- **Implementation:** `xtask/src/checks/ctx001_file_length.rs`.
- **Diagnostic code:** `CTX001`.
- **Promotion path:** if a future macro can structurally enforce module
  decomposition (e.g. a `#[bounded_module]` attribute that rejects oversized
  files at expansion time), this rule moves to tier 2 and the xtask check is
  retired.
